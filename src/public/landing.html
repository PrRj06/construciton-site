Frequently Asked Questions

Constructor (site worker) can submit daily updates, view dashboards, and browse all updates. They cannot delete records. Contractor (project manager) has full access including the ability to delete records, edit all submissions, and generate reports.

Yes. SiteTrack is designed for multi-site management. The Site Progress Tracker section gives you a visual overview of all active sites, and you can filter updates by site name or date range from the All Updates section.

SiteTrack accepts any standard image format (JPG, PNG, WEBP, etc.). Files must be under 5MB per upload (4MB on cloud deployment). You'll see a preview before submitting, and photos can be viewed in a full-screen lightbox from the updates table.

Absolutely. The entire application is responsive and designed for use on mobile phones, tablets, and desktops. Field workers can log updates directly from the construction site using any smartphone browser.

Click the " Print Report" button in the top bar. SiteTrack will fetch all your update records, generate a formatted HTML table with all dates, sites, work descriptions, and progress percentages, and trigger your browser's print dialog. You can print or save as PDF.

All data is stored in MongoDB with timestamps for audit tracking. The platform is deployed on Vercel with environment-variable-based credential management. File uploads are handled server-side and served via secure URLs.
